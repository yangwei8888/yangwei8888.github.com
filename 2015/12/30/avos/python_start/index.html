使用 virtualenv

如果您不需要使用 virtualenv，可以跳过这一步。

使用 virtualenv 可以创建一个与系统隔离的 Python 环境，在其中安装的第三方模块版本不会与系统自带的或者其他项目中的模块冲突。

之后您在当前 shell 中安装的第三方模块，都只会保存在当前项目目录下。

virtualenv 的更详细使用方法请参考官方文档。
